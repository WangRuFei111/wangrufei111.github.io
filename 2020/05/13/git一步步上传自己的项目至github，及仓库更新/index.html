git一步步上传自己的项目至github，及仓库更新

  • 2020-05-13

转载，亲测有效

仓库更新

  1. 首先切换到你的本地仓库中更新的项目目录
  2. 可以在用Git Bash用cd命令切换，也可以直接右键项目点击git bash here
  3. git status //查看仓库状态
  4. git add * //用于更新所有代码
  5. git commit -m “” //提交的备注
  6. git push origin master //将本地仓库的代码更新到github的仓库

分支：

1、查看当前库中有哪些分支：git branch
2、创建一个分支：git branch version1.0

​ 再次输入git branch查看分支信息

3、切换分支：git checkout version1.0
4、删除分支：git branch -D version1.0

需要注意的是：在version1.0分支上修改并提交的代码将不会影响到master分支，同理。如果我们在version1.0分支上修改一个bug，在master分支上这个bug仍然是存在的，这时将修改的代码一行行复制到master分支上显然不是一种聪明的做法，最好的办法就是使用merge命令来完成合并操作。

git checkout master
git master version1.0

这样两行命令，就可以把version1.0分支上修改并提交的内容合并到master分支上，当然，在合并的时分支的时候还有可能出现代码冲突的情况，这个时候你就需要静下心来慢慢地找出并解决这些冲突了。

与远程版本库协作：

如何使用Git来进行团队合作开发呢？这就需要有一个远程地版本库，团队地每一个成员都从这个版本库中获取到最原始地代码，然后各自开发，并且以后每次提交地代码都同步到远程版本库上就可以了。另外，团队中地每个成员最好都要养成经常从版本库中获取最新代码地习惯，不然地话，大家的代码就很可能经常出现冲突。

比说现在又一个远程版本库地Git地址是https://github.com/WangRuFei111/fei99.github.io.git，就可以使用如下地命令将代码下载到本地：

git clone https://github.com/WangRuFei111/fei99.github.io.git

之后如果你在这份代码地基础上进行了一些修改和提交，就需要借助pull命令来完成了：

git push origin master

其中，origin部分指定的是远程库版本的Git地址，master部分指定的是同步到哪一个分支上，上述的命令就完成了将本地代码同步到https://github.com/WangRuFei111/fei99.github.io.git这个版本库的master分支上的功能。

如何将远程版本库上的修改同步到本地。Git提供了两种命令来实现这个功能，分别是fetch和pull，fetch的语法规则和push差不多：

git fetch origin master

执行这个命令后，就会将远程版本库上的代码同步到本地，不过同步下来的代码并不会合并到任何分支上去，而是会存放一个origin/master分支上，这时我们可以通过diff命令来查看远程版本库上到底修改了那些东西：

git diff origin/master

之后在调用merge命令将origin/master分支上的修改合并到主分支上即可：

git merge origin/master

而pull命令是相当于将fetch和merge这两个命令放在一起执行了，它可以从远程版本库上获取最新的代码并且合并到本地：

git pull origin master
